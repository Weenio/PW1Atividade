API (Application Programming Interface)

O Básico

A API é um conjunto de regras e padrões que define como diferentes componentes de software podem interagir uns com os outros. As APIs fornecem um conjunto de métodos, funções e protocolos para acessar recursos e funcionalidades específicas de um sistema, como serviços web, bancos de dados, sistemas operacionais e muito mais.

As APIs podem ser usadas para gerar interação entre recursos de terceiros em um aplicativo, como autenticação de usuário, processamento de pagamentos e interação com banco de dados.

É comum se confundir, pensando que a API é a junção e a boa função entre HTML, CSS e JavaScript, mas isso está errado. Embora o HTML, CSS e JavaScript sejam fundamentais, a API é uma camada adicional de interação e comunicação com outros sistemas e recursos, além de oferecer funcionalidades específicas fornecidas por esses sistemas. Todavia, isso não implica a interação entre eles. Quando se trata de interagir com APIs, o JavaScript é a linguagem principal usada para fazer solicitações HTTP para um servidor, receber e enviar dados e processar as respostas. Essas solicitações e respostas normalmente são formatadas em JSON (JavaScript Object Notation), que é uma forma comum de estruturar dados para transmissão entre um cliente e um servidor. O HTML e o CSS também ajudam, sendo ultilizados para exibir e estilizar os dados obtidos das APIs. O HTML é usado para estruturar e organizar os elementos da página, enquanto o CSS é usado para aplicar estilos e formatar esses elementos.

Principais usos

Como dito antes, API é um conjunto de protocolos que permite a interação e comunicação entre diferentes softwares. Ela define a forma como os componentes de software devem se comunicar e interagir uns com os outros, permitindo o acesso a funcionalidades e recursos específicos de um certo sistema.

Elas podem ser encontradas em uma variedade de contextos, incluindo sistemas operacionais, aplicativos, frameworks e serviços web. Elas fornecem uma interface consistente e padronizada para que os desenvolvedores ultilizem os recursos e funcionalidades oferecidos por um determinado software, sem precisar conhecer os detalhes internos de implementação.

As APIs podem oferecer diferentes tipos de funcionalidades, como acesso a banco de dados, integração com serviços externos, manipulação de arquivos, interação com hardware, processamento de pagamentos, autenticação de usuários, entre outros. Por essa página ser simples, não temos nenhum exemplo prático do código, mas normalmente, esses elementos não são programados dentro da página HTML, e a interação é feita por um script em JavaScript. Essa interação gera, como antes dito, ou um arquivo em JSON ou XML.

No geral, a API é como se fosse o garçom que faz a ligação do front-end, com o back-end.
